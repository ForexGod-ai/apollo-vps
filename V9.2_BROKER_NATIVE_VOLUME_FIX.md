## 🎓 KEY LESSONS LEARNED

### **1. Never Hardcode Instrument-Specific Values**:

**BAD**:
```csharp
if (symbol == "BTCUSD") volume = 50000;  // ❌ What if broker changes config?
if (symbol == "EURUSD") volume = 100000; // ❌ Not scalable
```

**GOOD**:
```csharp
long volume = (long)symbol.QuantityToVolumeInUnits(lotSize);  // ✅ Dynamic
```

### **2. Always Use Broker APIs**:

cTrader provides:
- `QuantityToVolumeInUnits()` - Lots → Units
- `VolumeInUnitsToQuantity()` - Units → Lots
- `NormalizeVolumeInUnits()` - Round to valid step
- `VolumeInUnitsMin/Max/Step` - Broker constraints

**Don't reinvent the wheel - use the API!**

### **3. Test with Actual Broker Constraints**:

Before going live:
1. Query broker's min/max/step for the symbol
2. Test edge cases (min volume, max volume, decimal lots)
3. Verify rounding behavior
4. Check different account types (Standard, ECN, etc.)

### **4. Log Everything During Volume Calculation**:

```csharp
Print($"Requested: {lotSize} lots");
Print($"Converted: {volumeInUnits} units");
Print($"Broker limits: {min}-{max} (step={step})");

```

private long CalculateUniversalVolume(TradeSignal signal, Symbol symbol)

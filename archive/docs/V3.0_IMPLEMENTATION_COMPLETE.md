#### 1. Status Flow: MONITORING → READY
```
MONITORING: Setup detected but NOT safe for execution
- Daily CHoCH/BOS found

- Missing: 4H CHoCH OR price not in FVG

READY: All conditions met, can execute
- ✓ 4H CHoCH in last 30 candles
- ✓ CHoCH direction = Daily trend
- ✓ CHoCH break_price WITHIN FVG
- ✓ Current price IN FVG zone
```

#### 2. Expanded 4H Lookback Window
```python
# OLD: Only last 10 candles
for h4_choch in reversed(h4_chochs[-10:]):
```

#### 3. Execution Blocker
```python
def execute_trade(..., status: str = "READY"):
    if status != 'READY':
        logger.warning("⛔ EXECUTION BLOCKED")
        return False
    # Continue...
```

**Impact:**
- Prevents premature entries during aggressive pullbacks

## 🔄 MONITORING → READY FLOW

### Daily Scan (00:05)
```
daily_scanner.py
   └─> smc_detector.py: Detect Daily CHoCH + FVG
       └─> Check 4H CHoCH (last 30 candles)
       └─> Check price in FVG
       └─> Assign status: READY or MONITORING
       └─> Save to monitoring_setups.json
       └─> Telegram notification
```

### Continuous Monitoring (Every 30s)
```
setup_executor_monitor.py
   └─> Load monitoring_setups.json
   └─> FOR EACH setup:
       ├─> IF status == 'MONITORING':
       │   └─> Re-check conditions (4H data updated?)
       │   └─> Update to READY if conditions met
       │
       └─> IF status == 'READY':
           └─> Check if price hit entry
           └─> execute_trade(status='READY')
           └─> ctrader_executor validates status again
           └─> Write to signals.json
           └─> cBot executes
```

### Execution Validation (Multi-Layer)
```
Layer 1: setup_executor_monitor.py
   └─> Check status == 'READY' before calling execute_trade()

Layer 2: ctrader_executor.py
   └─> Validate status parameter
   └─> Block if status != 'READY'
   └─> Log warning + reason

Layer 3: PythonSignalExecutor cBot
   └─> Check for duplicate positions
   └─> Execute trade in cTrader
```

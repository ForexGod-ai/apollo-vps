# V3.0 STRICT ENTRY CONFIRMATION SYSTEM

## 🎯 OBIECTIV
Prevenim intrări premature în timpul pullback-urilor agresive prin implementarea unui sistem strict de confirmare pe 4H înainte de execuție.

---

## 📋 STATUS FLOW: MONITORING → READY

### 1️⃣ MONITORING Status
**Definiție:** Setup detectat pe Daily, dar NU este încă safe pentru execuție.

**Condiții pentru MONITORING:**
- ✅ Daily CHoCH detectat (trend change) SAU Daily BOS (trend continuation)
- ✅ FVG valid găsit (gap ≥ 0.10%, body dominance ≥ 25%)
- ❌ NU există 4H CHoCH în ultima fereastră (30 candles) SAU
- ❌ Prețul NU este în zona FVG

**Acțiune:**
- Setup salvat în `monitoring_setups.json` cu `status: "MONITORING"`
- Telegram notification: `⏳ MONITORING`
- **EXECUȚIA BLOCATĂ** - așteptăm confirmări

---

### 2️⃣ READY Status
**Definiție:** Setup complet confirmat, poate fi executat automat.

**Condiții pentru READY (TOATE SIMULTAN):**

#### A. Confirmare 4H CHoCH
```
✓ Există CHoCH pe 4H în ultima fereastră (30 candles = ~5 zile)
✓ Direcția CHoCH = Direcția trend-ului Daily
✓ CHoCH break_price este ÎNĂUNTRUL zonei FVG (fvg.bottom ≤ break_price ≤ fvg.top)
```

**Exemplu BULLISH:**
```python
Daily: CHoCH bullish @ 1.0850
FVG: 1.0800 - 1.0820
4H: Trebuie să găsim CHoCH bullish cu break_price între 1.0800-1.0820 (în ultimele 30 candles)
```

#### B. Preț în FVG
```
✓ current_price trebuie să fie ÎNĂUNTRUL FVG zone
✓ BULLISH: fvg.bottom ≤ current_price ≤ fvg.top
✓ BEARISH: fvg.bottom ≤ current_price ≤ fvg.top
```

**Validare:**
```python
def is_price_in_fvg(current_price, fvg):
    return fvg.bottom <= current_price <= fvg.top
```

**Acțiune:**
- Setup actualizat în `monitoring_setups.json` cu `status: "READY"`
- Telegram notification: `✅ READY TO EXECUTE`
- **EXECUȚIA PERMISĂ** - `setup_executor_monitor.py` poate trimite signal

```
1. daily_scanner.py scans all pairs at 00:05

   → status = 'READY'

   → Telegram: "✅ READY TO EXECUTE"
ELSE:
   → status = 'MONITORING'
   → Save to monitoring_setups.json
   → Telegram: "⏳ MONITORING (waiting for confirmation)"
```

### Scenario 2: Monitoring Update (Every 30s)
```
setup_executor_monitor.py runs every 30s:

FOR EACH setup in monitoring_setups.json:
   IF status == 'MONITORING':
      → Re-check 4H data (new candles?)
      → Re-check price position (entered FVG?)
      → Update status if conditions met
      → Save updated setup
   
   IF status == 'READY':
      → Check if price hit entry
      → IF YES:
         → execute_trade() (will validate status again)
         → Write to signals.json
         → cBot executes automatically
```

```
ctrader_executor.execute_trade() called:

1. CHECK status parameter

      → LOG warning
      → RETURN False (BLOCKED)
   
2. IF status == 'READY':
   → Calculate pips
   → Write signal to signals.json
   → cBot reads and executes
```

## 🔍 4H LOOKBACK WINDOW

### V2.1 (Old)
```python
# Only checked last 10 candles (too short!)
for h4_choch in reversed(h4_chochs[-10:]):
    ...
```

### V3.0 (New)
```python
# Check last 30 candles (~5 days of 4H data)
recent_h4_chochs = [ch for ch in h4_chochs if ch.index >= len(df_4h) - 30]

for h4_choch in reversed(recent_h4_chochs):
    if h4_choch.direction != current_trend:
        continue
    if fvg.bottom <= h4_choch.break_price <= fvg.top:
        valid_h4_choch = h4_choch
        break
```

**Beneficiu:** Găsim CHoCH-uri mai îndepărtate care confirmă trend-ul, nu doar cele foarte recente.

  • FIFO Processing: First-in-first-out execution order
  • Atomic Writes: Temp file + atomic rename (race-condition proof)

ctrader_executor.py Threading Queue os.replace()
🔐 Atomic File Protocol
Zero-data-loss write mechanism using temporary files and atomic filesystem operations:
1. Create temp file: /signals.json.tmp 2. Write JSON data to temp (buffered) 3. Force OS write: fsync() 4. Atomic rename: os.replace(temp, signals.json) └─→ Cannot be interrupted mid-operation 5. C# cBot reads signals.json (always complete) 6. Writes confirmation.json with execution details

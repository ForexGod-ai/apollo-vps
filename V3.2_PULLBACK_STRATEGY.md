# 🎯 GLITCH V3.2 - PULLBACK STRATEGY

**Enhanced Entry System with 50% Fibonacci Retracement**

---

## 📊 STRATEGY OVERVIEW

V3.2 introduces a **pullback-based entry system** that waits for price to retrace to the 50% Fibonacci level after a CHoCH confirmation before executing trades. This approach significantly improves entry quality, reduces stop loss distance, and increases risk:reward ratios.

### Key Improvement
Instead of entering immediately after 1H CHoCH confirmation, we wait for price to **pullback to 50% Fibo level** of the CHoCH swing, then enter with **SL below the last swing low/high**.

### **V3.2 PULLBACK FLOW:**
```

2. Setup Executor detects 1H CHoCH in FVG zone
3. ⏳ WAIT for pullback to 50% Fibo level
4. ✅ ENTRY when price reaches Fibo 50% (±10 pips tolerance)
5. SL placed below last swing low/high (3-5 candles)
6. Wait 48h for 4H CHoCH to scale in Entry 2

✅ BENEFITS:
- Entry at better price (mid-pullback)
- Smaller SL distance (30-40% reduction)
- Higher probability of success (avoids fakeouts)
- Improved R:R ratios (1:15-20 average)
```

## 🎯 ENTRY RULES

### **1. CHoCH Detection**
- Must have valid 1H CHoCH in direction of Daily CHoCH
- CHoCH must occur within FVG zone (±10 pips tolerance)
- Minimum swing size: 30 pips (filters noise)

### **2. Fibonacci Calculation**
```python
# For BULLISH setup:
choch_low = min(last 3-5 candles)
choch_high = CHoCH candle high
fibo_50 = choch_low + (choch_high - choch_low) * 0.5

# For BEARISH setup:
choch_high = max(last 3-5 candles)
choch_low = CHoCH candle low
fibo_50 = choch_high - (choch_high - choch_low) * 0.5
```

### **3. Pullback Wait**
- Wait for price to retrace to Fibo 50% level
- Tolerance: ±10 pips (adjustable per pair volatility)
- Timeout: 24 hours from CHoCH detection
- If timeout: either skip setup or enter at current price (configurable)

### **4. Stop Loss Placement**
```python
# For BULLISH setup:
swing_low = min(last 3-5 candles before entry)
stop_loss = swing_low - (10 pips buffer)

# For BEARISH setup:
swing_high = max(last 3-5 candles before entry)
stop_loss = swing_high + (10 pips buffer)
```

### **5. Entry Execution**
- Entry 1: 50% of position size when Fibo 50% reached
- Wait 48h for 4H CHoCH confirmation
- Entry 2: 50% scale-in if 4H CHoCH detected
- If no 4H CHoCH in 48h: manage Entry 1 based on P&L

```python
def calculate_choch_fibonacci(df_h1: pd.DataFrame, choch_idx: int, direction: str) -> dict:
    """
    Calculate 50% Fibonacci retracement level after CHoCH
    
    Args:
        df_h1: 1H timeframe dataframe
        choch_idx: Index of CHoCH candle
        direction: 'bullish' or 'bearish'
    
    Returns:
        {
            'fibo_50': float,
            'swing_high': float,
            'swing_low': float,
            'swing_range': float
        }
    """
    lookback = 5  # Check last 5 candles
    
    if direction == 'bullish':
        swing_low = df_h1.iloc[choch_idx-lookback:choch_idx+1]['low'].min()
        swing_high = df_h1.iloc[choch_idx]['high']
        fibo_50 = swing_low + (swing_high - swing_low) * 0.5
        
    elif direction == 'bearish':
        swing_high = df_h1.iloc[choch_idx-lookback:choch_idx+1]['high'].max()
        swing_low = df_h1.iloc[choch_idx]['low']
        fibo_50 = swing_high - (swing_high - swing_low) * 0.5
    
    return {
        'fibo_50': round(fibo_50, 5),
        'swing_high': round(swing_high, 5),
        'swing_low': round(swing_low, 5),
        'swing_range': round(abs(swing_high - swing_low), 5)
    }
```

#### **2. `validate_pullback_entry()` in smc_detector.py**
```python
def validate_pullback_entry(df_h1: pd.DataFrame, fibo_data: dict, direction: str, tolerance_pips: int = 10) -> dict:
    """
    Check if current price has pulled back to Fibo 50% level
    
    Args:
        df_h1: 1H timeframe dataframe
        fibo_data: Dictionary from calculate_choch_fibonacci()
        direction: 'bullish' or 'bearish'
        tolerance_pips: Acceptable distance from Fibo 50% (default 10)
    
    Returns:
        {
            'pullback_reached': bool,
            'current_price': float,
            'distance_to_fibo': float (in pips),
            'stop_loss': float,
            'entry_price': float
        }
    """
    current_price = df_h1.iloc[-1]['close']
    fibo_50 = fibo_data['fibo_50']
    
    # Convert tolerance to price units (assuming 5-digit quotes)
    tolerance = tolerance_pips * 0.0001  # For JPY pairs: 0.01
    
    distance = abs(current_price - fibo_50)
    pullback_reached = distance <= tolerance
    
    # Calculate SL below last swing low/high
    if direction == 'bullish':
        swing_low = df_h1.iloc[-5:]['low'].min()  # Last 5 candles
        stop_loss = swing_low - (10 * 0.0001)  # 10 pips buffer
    else:
        swing_high = df_h1.iloc[-5:]['high'].max()
        stop_loss = swing_high + (10 * 0.0001)
    
    return {
        'pullback_reached': pullback_reached,
        'current_price': round(current_price, 5),
        'distance_to_fibo': round(distance / 0.0001, 1),  # in pips
        'stop_loss': round(stop_loss, 5),
        'entry_price': round(fibo_50, 5) if pullback_reached else None
    }
```

## 🚨 RISKS & CONSIDERATIONS

### **Potential Issues:**

1. **Missed Entries:**
   - Some setups may never pullback to Fibo 50%
   - Solution: Configurable timeout with force-entry option

2. **Whipsaws:**
   - Price may hit Fibo 50% multiple times
   - Solution: Only take first touch, require bullish/bearish candle close

3. **Overnight Gaps:**
   - Price may gap past Fibo 50% level
   - Solution: Accept entries within tolerance even if gapped

4. **Low Volatility:**
   - Some pairs may not pullback enough
   - Solution: Adjust tolerance per pair volatility

### **Mitigation Strategies:**

1. **Dynamic Tolerance:** Adjust ±10 pips based on pair ATR
2. **Candle Confirmation:** Require bullish/bearish close at Fibo 50%
3. **Timeout Options:** Configurable skip/force-entry/wait-longer
4. **Pair-Specific Settings:** Different parameters for GBP vs EUR pairs

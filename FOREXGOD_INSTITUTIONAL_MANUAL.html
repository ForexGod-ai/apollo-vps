Watchdog V4.0 — Self-Healing Protocol

The immutable guardian daemon that ensures absolute system continuity. Monitors all critical processes via PID verification every 60 seconds, automatically resurrecting any process that deviates from nominal operation. Operates as the root sentinel — if everything else fails, Watchdog rebuilds.
▾
watchdog_monitor.py 415 lines Guardian Daemon
Self-healing process supervisor that maintains a registry of all critical runtime daemons. Performs PID liveness checks via os.kill(pid, 0) signal probing, automatic subprocess resurrection with full argument preservation, and Telegram alerting on any process anomaly. Implements graceful degradation — individual module failure never cascades to system-wide shutdown.

MTF Sniper — D1-4H-1H Synchronicity Protocol

The precision execution pipeline that converts analytical setups into live market positions. Implements the V10.4 D1 Bias + 4H Sync architecture: entries occur only when D1 bias, 4H CHoCH body closure, and H4 sync FVG all align with surgical precision.

setup_executor_monitor.py 1,991 lines Execution Engine
The V10.4 execution brain. Monitors monitoring_setups.json every 30 seconds, validates pullback entries into the H4 sync FVG (with Daily FVG fallback), manages 4H structure lock with live CHoCH detection, and executes multi-entry scale-in sequences. The Python→C# signal bridge. Risk Layer · Layer 3

Unified Risk Shield — Deep Sleep & Atomic Locking

The non-negotiable risk management layer that sits between analysis and execution. Every trade must pass through the Risk Shield before reaching the broker. Implements daily loss limits, position caps, duplicate guards, and dynamic lot calculation with zero tolerance for bypass.
▾
unified_risk_manager.py 788 lines Risk Engine
Centralized risk validation engine. Enforces daily loss threshold (percentage-based), maximum concurrent position limits, per-pair duplicate detection, and risk-per-trade lot calculation using ATR-adjusted volatility metrics. Implements "Deep Sleep" mode — when daily loss exceeds threshold, the entire execution pipeline enters dormancy until the next trading session. All file operations use atomic writes with fcntl advisory locks.
Daily Loss Threshold Position Limits Duplicate Guard ATR Lot Sizing Deep Sleep Protocol Atomic File I/O fcntl Advisory Locks

Technology Stack
Engineering Foundation
The complete technology stack powering the POCOVNICU Terminal — chosen for reliability, performance, and zero-dependency deployment.

Core Runtime

Python 3.14

Analytical engine, risk management, daemon orchestration. Latest Python release for maximum performance with match-case, type hints, and asyncio patterns.

Broker Execution

C# .NET · cTrader cBots

Four custom cBots running in cTrader's .NET runtime for direct broker API access, order execution, position management, and market data streaming.

IPC Protocol

Atomic JSON + fcntl Locks

All inter-process communication uses atomic write-rename patterns with POSIX fcntl advisory locks. Zero data corruption across concurrent access.

Process Management

macOS launchd

Native daemon management with KeepAlive policies, boot-time initialization, standard output/error log routing, and automatic restart on failure.

Database

SQLite 3 · WAL Mode

Trade history persistence with Write-Ahead Logging for concurrent read access. Daily automated backups with retention policy.

Communication

Telegram Bot API

Bidirectional operator interface with command processing, rich notifications, chart image delivery, and remote system control capabilities.

Market Data

HTTP Caching + cTrader Feed

Dual data sourcing: HTTP-cached OHLCV data for analysis, real-time cTrader market data stream for execution-grade pricing.

Charting

matplotlib + Custom Renderer

Proprietary chart generation engine producing annotated candlestick charts with SMC overlays, FVG zones, and CHoCH markers for Telegram delivery.

Strategy Engine

Glitch in Matrix SMC

Proprietary Smart Money Concepts implementation: D1 bias lock, 4H CHoCH sync, FVG entry targeting, ATR prominence filtering, and institutional order flow detection.

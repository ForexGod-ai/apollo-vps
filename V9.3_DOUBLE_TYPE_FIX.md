## 🚨 THE SILENT KILLER: TYPE TRUNCATION

### **V9.2 Fatal Flaw**:

```csharp
// V9.2 (WRONG - TRUNCATION BUG):
double lotSize = 0.50;
long forcedVolume = (long)btcSymbol.QuantityToVolumeInUnits(lotSize);
//                  ^^^^^^
//                  Cast to long TRUNCATES decimals!
//
// Flow:
// 1. QuantityToVolumeInUnits(0.50) returns → 0.50 (double)
// 2. (long) cast converts → 0 (integer truncation)
// 3. ExecuteMarketOrder(0) → ❌ REJECTED: Volume = 0
```

### **The C# Type System Trap**:

| Expression | Type | Value | Result |
|------------|------|-------|--------|
| `QuantityToVolumeInUnits(0.50)` | `double` | `0.50` | ✅ Correct |
| `(long)0.50` | `long` | `0` | ❌ Truncated! |
| `(int)0.99` | `int` | `0` | ❌ Truncated! |
| `(long)1.50` | `long` | `1` | ❌ Truncated! |

**Key Insight**: C# does NOT round when casting `double → long`. It **truncates** (cuts off decimals).

### **The Fix**:

### **Why This Works**:

1. **`QuantityToVolumeInUnits()`** returns `double` (not `long`)
2. **Keep as `double`** throughout the calculation chain
3. **`NormalizeVolumeInUnits()`** accepts and returns `double`
4. **`ExecuteMarketOrder()`** accepts `double` for volume parameter
5. **Broker supports decimal volumes**: 0.01, 0.50, 1.25, etc.

## 🎓 KEY LESSONS LEARNED

### **1. Never Cast Decimals to Integers for Financial Data**:

**BAD**:
```csharp
long volume = (long)0.50;        // 0 - Lost money!
int price = (int)1.234;          // 1 - Lost precision!
long amount = (long)balance;     // Truncated - Wrong balance!
```

**GOOD**:
```csharp
double volume = 0.50;            // 0.50 - Preserved!
decimal price = 1.234m;          // 1.234 - Full precision!
double amount = balance;         // Exact value!
```

### **2. C# Casting Behavior**:

```csharp
// Integer casts TRUNCATE (do not round!)
(int)0.1    // 0
(int)0.5    // 0
(int)0.9    // 0
(long)1.5   // 1
(long)1.9   // 1

// If you need rounding, use Math.Round()
Math.Round(0.5, MidpointRounding.AwayFromZero)  // 1
Math.Round(1.5, MidpointRounding.AwayFromZero)  // 2
```

### **3. cTrader API Type Signatures**:

```csharp
// These methods work with DOUBLE (not long!)
double QuantityToVolumeInUnits(double quantity);
double NormalizeVolumeInUnits(double volume, RoundingMode mode);
TradeResult ExecuteMarketOrder(TradeType type, string symbol, double volume, ...);

// Properties are also DOUBLE
double VolumeInUnitsMin { get; }
double VolumeInUnitsMax { get; }
double VolumeInUnitsStep { get; }
```

**Key Insight**: API designed for DECIMAL volumes. Don't fight it with integer types!

## 🔮 FUTURE-PROOFING

### **Type Safety Guidelines**:

1. **Financial volumes**: Always use `double` or `decimal`
2. **Prices**: Use `double` (cTrader standard)
3. **Money amounts**: Use `decimal` for maximum precision
4. **Counts/IDs**: Use `int` or `long` (whole numbers only)

### **Recommended Pattern**:

```csharp
// Universal volume handler
private double CalculateSafeVolume(TradeSignal signal, Symbol symbol)
{
    // Step 1: Get lot size (with fallback)
    double lotSize = signal.LotSize > 0 ? signal.LotSize : 0.01;
    
    // Step 2: Convert to broker units (KEEP AS DOUBLE)
    double volumeInUnits = symbol.QuantityToVolumeInUnits(lotSize);
    
    // Step 3: Normalize to broker step
    volumeInUnits = symbol.NormalizeVolumeInUnits(volumeInUnits, RoundingMode.Down);
    
    // Step 4: Enforce broker limits
    if (volumeInUnits < symbol.VolumeInUnitsMin)
        volumeInUnits = symbol.VolumeInUnitsMin;
    if (volumeInUnits > symbol.VolumeInUnitsMax)
        volumeInUnits = symbol.VolumeInUnitsMax;
    
    // Step 5: NEVER cast to long!
    return volumeInUnits;  // ✅ Return as double
}
```

## 🎓 LESSONS LEARNED

1. **Never use exact string matching for symbols**:
   - Scanner might return different formats
   - API responses vary (uppercase, lowercase, with/without slashes)
   - User input unpredictable

2. **Always normalize before comparison**:
   - `.upper()` → Case-insensitive
   - `.replace(' ', '')` → Whitespace-proof
   - `.replace('/', '')` → Separator-proof

3. **Use flexible matching**:
   - `'BTC' in symbol_clean` better than `symbol == 'BTCUSD'`
   - Catches partial matches
   - Future-proof for API changes

4. **Log actual values, not assumptions**:
   - Old: `"Forcing 0.50 lots for BTCUSD"` (hardcoded)
   - New: `"Forcing 0.50 lots for {symbol}"` (actual value)
   - Helps debugging formatting issues
